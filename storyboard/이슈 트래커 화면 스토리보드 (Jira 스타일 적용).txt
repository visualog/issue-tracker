이슈 트래커 화면 스토리보드 (Jira 스타일 적용)
1. 로그인 화면
* (변경 없음) 화면 중앙 로그인 폼 유지. AD 계정으로 인증.
2. 메인 화면 (로그인 후 공통 레이아웃)
* 레이아웃 구성:
   * 상단 헤더: 서비스 로고, 전역 검색창, (MIS/Admin) '만들기'(이슈 생성) 버튼, 알림 아이콘, 사용자 프로필/로그아웃 드롭다운.
   * 좌측 사이드바 (고정 네비게이션): 역할에 따라 메뉴 항목 표시 (아래 각 사용자별 설명 참조).
   * 메인 컨텐츠 영역: 사이드바에서 선택된 메뉴에 해당하는 내용 표시 (목록, 보드, 캘린더, 상세 정보 등).
3. 메인 화면 - 컨텐츠 영역 (일반 사용자)
* 사이드바 메뉴:
   * 내 팀 요청 목록 (기본 선택)
   * 칸반 보드 뷰 (내 팀)
   * 캘린더 뷰 (내 팀)
* 메인 컨텐츠 영역 (기본: 내 팀 요청 목록):
   * 상단: '내 팀 요청 현황' 타이틀, (필요시 간단한 필터 옵션 - 예: 상태별)
   * 중앙: 요청 목록 테이블 (읽기 전용). 컬럼 예시: Key (고유 ID), 요청명, 상태, 담당자, 요청일. 각 행 클릭 시 '6. 요청 상세 조회 모달' 표시.
* 주요 기능:
   * 사이드바 메뉴를 통해 뷰 전환 (목록, 칸반, 캘린더).
   * 목록에서 특정 요청 클릭하여 상세 정보 확인 (모달).
   * (칸반/캘린더 뷰) 시각화된 요청 현황 확인 (읽기 전용).
4. 메인 화면 - 컨텐츠 영역 (팀장급 / MIS / Super Admin)
* 사이드바 메뉴:
   * 모든 요청 목록 (기본 선택)
   * 칸반 보드 뷰 (전체)
   * 캘린더 뷰 (전체)
   * (MIS/Admin) 내 담당 요청
   * (MIS/Admin) 월별 통계/리포트
   * (MIS) 마이 페이지 (개인 일정 관리 등)
   * (Super Admin) 관리자 설정
* 메인 컨텐츠 영역 (기본: 모든 요청 목록):
   * 상단: '전체 요청 현황' 타이틀, 고급 필터 바 (요청 부서, 담당자, 상태, 작업명 등 다중 조건 필터링/검색, 필터 저장 기능 - 선택적).
   * 중앙: 요청 목록 테이블 (팀장: 읽기 전용, MIS/Admin: 상호작용 가능). 컬럼 예시: Key, 요청명, 상태, 담당자, 요청자, 요청일, 마감(예정)일. 각 행 클릭 시 '6. 요청 상세 조회 모달' 표시.
* 주요 기능:
   * 사이드바 메뉴를 통해 뷰/기능 전환.
   * 고급 필터링/검색으로 원하는 요청 조회.
   * (MIS/Admin) 헤더의 '만들기' 버튼 클릭 -> '7. 새 요청 생성 모달' 표시.
   * (MIS/Admin - 칸반 뷰) 드래그 앤 드롭으로 상태 변경.
   * 목록/보드/캘린더에서 특정 요청 클릭 -> '6. 요청 상세 조회 모달' 표시.
5. (삭제됨) - 홈 대시보드 개념은 '메인 화면' 레이아웃과 컨텐츠 영역으로 통합됨.
6. 요청 상세 조회 모달
* 화면 명: 요청 상세 정보 (모달)
* 트리거: 목록/보드/캘린더에서 특정 요청 클릭 시
* 레이아웃 구성: 화면 위에 오버레이되는 모달 창 또는 우측 패널 형태.
* 주요 구성 요소:
   * 헤더: Key (예: INFO-123), 요청 제목. (MIS/Admin) 상태 변경 드롭다운/버튼, 편집/삭제 등 액션 버튼. 닫기 버튼(X).
   * 핵심 정보: 담당자, 요청자, 요청 부서, 생성일, 수정일, 마감(예정)일 등.
   * 링크 영역 (강조):
      * Co 프로젝트 링크 버튼 (매우 중요): "Co에서 세부 내용 보기/작업" 레이블.
      * Git 링크, Figma 링크 (아이콘 버튼 형태 가능).
   * 탭 또는 섹션:
      * 설명: 초기 요청 내용.
      * 댓글: 댓글 목록 (2단계 트리), 댓글 입력 필드.
      * (선택적) 히스토리: 상태 변경, 담당자 변경 등 활동 기록.
* 주요 기능:
   * 상세 정보 확인.
   * Co/Git/Figma 링크 클릭 (새 탭 이동).
   * 댓글 조회/등록/수정(본인)/삭제(본인). (등록/수정 시 담당자 알림).
   * (MIS/Admin) 상태 변경, 정보 수정 ('8. 요청 수정 모달' 트리거 또는 인라인 편집), 요청 삭제.
7. 새 요청 생성 모달
* 화면 명: 새 요청 만들기 (모달)
* 트리거: (MIS/Admin) 헤더의 '만들기' 버튼 클릭 시
* 레이아웃 구성: 모달 창 형태.
* 주요 구성 요소:
   * 모달 헤더: "새 요청 만들기" 타이틀, 닫기 버튼(X).
   * 입력 폼:
      * 요청 제목 (필수)
      * 요청 내용/설명 (Rich Text Editor 사용 가능)
      * 요청 부서 (선택)
      * 요청자 지정 (필수, 검색/선택 UI)
      * 담당자 지정 (필수, 검색/선택 UI)
      * Co 프로젝트 링크 (필수)
      * Git 프로젝트 링크
      * Figma 링크
      * (선택적) 초기 상태, 마일스톤/예상일 등
   * 하단 버튼: 만들기, 취소.
* 주요 기능:
   * 폼 입력.
   * '만들기' 클릭 -> 유효성 검사 후 저장, 관련자 알림, 모달 닫고 목록 새로고침.
   * '취소' 클릭 -> 모달 닫기.
8. 요청 수정 모달
* 화면 명: 요청 수정 (모달)
* 트리거: (MIS/Admin) 상세 조회 모달에서 '편집' 버튼 클릭 시
* 레이아웃 구성: '7. 새 요청 생성 모달'과 유사하나, 기존 데이터가 채워져 있음.
* 주요 구성 요소: '7'과 동일.
* 주요 기능:
   * 기존 정보 수정.
   * '저장' 클릭 -> 변경 사항 저장, 관련자 알림, 모달 닫고 상세 정보/목록 새로고침.
   * '취소' 클릭 -> 모달 닫기.
9. 마이 페이지 (MIS)
* (변경) 별도 페이지 대신, 사이드바의 '내 담당 요청' 메뉴와 '개인 일정 관리' 메뉴로 분리 고려.
   * 내 담당 요청: '4. 메인 화면 - 컨텐츠 영역'과 동일한 목록/보드/캘린더 뷰 제공 (자신이 담당자인 이슈만 필터링).
   * 개인 일정 관리: (신규 화면 필요) 개인적인 일정/메모를 등록하고 캘린더 형태로 볼 수 있는 기능. (Jira 기본 기능 아님)
10. 관리자 페이지 (Super Admin)
* 접근: 사이드바의 '관리자 설정' 메뉴 클릭.
* 레이아웃 구성: 좌측에 관리 하위 메뉴 (서비스 관리, 사용자 관리, 시스템 설정 등), 우측에 해당 설정 화면 표시. (Jira의 설정 페이지와 유사)
* 주요 구성 요소/기능: 기존과 동일 (서비스 관리, 대시보드 통계, 관리자 계정 관리).
이 Jira 스타일을 적용한 스토리보드가 실제 화면 설계에 더 구체적인 가이드를 제공할 수 있기를 바랍니다.
